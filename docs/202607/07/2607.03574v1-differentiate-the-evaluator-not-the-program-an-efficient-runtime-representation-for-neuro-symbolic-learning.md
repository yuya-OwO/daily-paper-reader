## 摘要
人工智能系统越来越多地提出可执行的科学模型，其价值既取决于符号结构，也取决于拟合的连续参数。这使得参数校准成为程序与参数联合搜索的瓶颈：外部循环可以生成数千个候选程序，但每个程序在评估前都需要内部基于梯度的优化。将每个候选程序分阶段编译成自己的可微图虽然使单个模型运行快速，但牺牲了保持搜索流畅性的“程序即数据”特性；基于解释器的方法将程序作为运行时数据保留，但需承担开销，且该开销主导了数值计算的工作量。我们提出了原生可微虚拟机（NDVM），这是一种运行时表示，它能够对可执行程序进行微分，而无需将每个候选程序编译成单独的图。NDVM将符号结构与可微数值状态分离：标签、符号、环境及控制流保留为原生运行时数据，而数值有效载荷则位于密集批处理缓冲区中，并沿实际执行轨迹记录精确的反向模式梯度，因此一次评估器的遍历可在大量参数向量上分摊成本。一个真实可微自托管Scheme解释器的锁定成本模型推动了该设计。我们实现了NDVM作为原生运行时，与参考后端具有前向和梯度等价性，每个通道约60倍的批量分摊，近线性多核扩展，以及两个独立前端。在由LLM提出的程序的固定预算联合搜索中，NDVM在挂钟时间上约快24倍达到高质量解，表明运行时微分可作为科学发现工作流的实用系统基础。

## Abstract
AI systems increasingly propose executable scientific models whose value depends on both their symbolic structure and their fitted continuous parameters. This makes parameter calibration the bottleneck of program-and-parameter co-search: an outer loop can generate thousands of candidate programs, but each needs an inner gradient-based optimization before it can be assessed. Staging each candidate into its own differentiable graph makes individual models fast but sacrifices the program-as-data property that keeps search fluid; interpreter-based approaches preserve programs as runtime data but pay interpreter overhead that dominates the numerical work.   We present the Native Differentiable Virtual Machine (NDVM), a runtime representation that differentiates executable programs without compiling each candidate into a separate graph. NDVM separates symbolic structure from differentiable numeric state: tags, symbols, environments, and control remain native runtime data, while numeric payloads live in dense batched buffers with exact reverse-mode gradients recorded along the realized execution trace, so one evaluator walk is amortized across large populations of parameter vectors. A locked cost model of a real differentiable self-hosted Scheme interpreter motivates the design. We realize NDVM as a native runtime with forward and gradient equivalence to the reference backend, about 60x per-lane batch amortization, near-linear multicore scaling, and two independent front ends. In fixed-budget co-search over LLM-proposed programs, NDVM reaches high-quality solutions about 24x sooner in wall-clock time, suggesting runtime differentiation as a practical systems foundation for scientific discovery workflows.

- **背景**：神经符号学习、可微科学建模等场景需要高效处理大量不同程序，参数校准成为瓶颈，而非程序生成或科学评估本身。现有方法无法同时满足搜索灵活性与执行效率。
- **意义**：作者通过锁定成本模型发现，在真实可微解释器中，前向时间的85-90%花费在值装箱（value boxing）和评估器遍历上，算术运算仅占1%或更少，且前向时间几乎与批次大小无关。这直接挑战了“性能瓶颈在数学计算”的常见假设，为设计新型运行时表示提供了依据。

## 2. 方法论

### 核心思想
- **区分评估器而非程序**：编译并微分一个固定的评估器，然后将任意程序作为数据输入该评估器运行，无需对每个程序进行逐程序的分阶段编译。程序保持为运行时数据，自动继承梯度。
- **结构/数值分离**：将运行时表示分为离散结构（标签、符号、地址、闭包、环境）和可微数值有效载荷。离散结构保留为原生标量数据，数值有效载荷放入密集批处理缓冲区。

### 关键技术细节
- **运行时值表示**：每个运行时值是一个紧凑标量记录 `⟨tag, aux, pid⟩`。`tag` 是整数类型标识，`aux` 保存直接量（符号ID、堆地址、闭包ID等），`pid` 索引数值有效载荷表（若为非数值则为哨兵）。非数值值不分配张量。
- **有效载荷表**：数值有效载荷存储在结构数组（SoA）表中，包含密集的原始（primal）和相邻（adjoint）缓冲区。有效载荷的第一个轴是批次轴B，因此一个标量结构值可指向形状为 [B] 或 [B, ...] 的批处理数值有效载荷。
- **控制流**：精确执行轨迹，通过原生指令流（switch分发或直接线程化）执行，不会对目标程序进行反编译。条件分支、数据相关分发、可变循环、尾调用均实现为虚拟机原语，而非宿主递归。
- **反向模式微分**：定义即运行的反向模式，仅对数值原语（加法、乘法、exp、log、矩阵乘法等）记录紧凑原生磁带；结构性操作（符号查找、标签测试、对分配、闭包构造、分支分发）不产生磁带节点。反向播放磁带时，通过向量化循环在批次轴上累积相邻。
- **批次作为第一类轴**：单次评估器遍历可处理多组参数向量，因为结构性遍历是共享的，仅有效载荷是宽的。支持命名轴（数据、重启、细胞、种群），并通过批次掩码处理数据相关的数值分支。当分支在种群中发散时，通过轨迹分桶将共享相同控制路径的通道分组。
- **堆、环境和符号**：由于所选子集排除突变，堆为一次性写入区域；环境使用关联列表，可选内联缓存；符号均内联为整数，避免字符串比较。

### 算法/流程示意（文字说明）
1. 前向传播：将程序作为数据输入编译好的NDVM评估器。评估器使用原生标量值进行结构操作（符号查找、条件判断、堆分配），遇到数值原语时，从当前执行轨迹记录一个磁带节点并操作批处理有效载荷。整个结构性遍历（值装箱、标签检查、环境查找）仅发生一次，不随批次大小缩放。
2. 反向传播：以磁带节点为索引，逆序回放，对每个数值原语应用向量-雅可比乘积（VJP），将梯度累积到相邻缓冲区中。磁带仅记录实际执行的数值操作，与解释器自身递归深度或程序大小无关。
3. 批处理：多个参数向量（或重启/细胞/数据点）共享同一个结构性遍历。非数值值（如条件分支的判别）在每个通道各自独立，但结构化决策（如分支选择）在通道间可能不同：分歧控制通过通道掩码实现，每个分支仅在其活跃通道上执行，相邻也仅门控到实际经过的通道。

## 6. 主要结论与发现

1. **瓶颈是表示而非算术**：在真实可微Scheme解释器中，前向时间的85-90%用于值装箱和评估器遍历，算术仅占1%或更少，且这一比例在2300倍成本范围内几乎不变。
2. **结构性/数值分离消除了装箱开销**：调优急切载荷-only表示相比标记急切基线快4-5x，但原生执行进一步增加8-2133x（取决于程序）。
3. **批处理摊销约60x**：将结构性遍历保持标量，仅拓宽有效载荷，使得每通道成本从B=1到256下降约60倍（通过PyTorch边界约21倍）。
4. **多核近线性扩展**：因候选之间无共享状态，使用线程本地NDVM在16核上达到约15x（93%效率）。
5. **精确梯度等价**：NDVM的磁带生成与基线后端完全相同的梯度，包括通过矩阵-伴随路径的卡尔曼噪音协方差梯度。差分测试通过600/600检查。
6. **联合搜索显著加速**：
   - 在标量回归任务中，NDVM在900秒内校准42,289个候选 vs 基线后端1,778个，23.8x吞吐量提升；达到R²>0.9快约24x（12.2秒 vs 288秒）。
   - 在递归重载任务中，吞吐量提升340x，NDVM成功恢复结构，基线后端未在预算内成功。
7. **NDVM在低重用联合搜索中胜出**：与手写JAX/分阶段图比较，转换点n⋆在213到42,185步之间，远大于联合搜索中每个候选的少量梯度步。结构缓存分阶段在约80次重用后超越NDVM，但联合搜索中每个骨架通常仅使用一次。

## 7. 优点

- **创新方法论**：提出“区分评估器而非程序”的设计哲学，从表示层面解决灵活性与效率的冲突，而非依赖传统编译或特种加速器。
- **系统实现完整**：NDVM被完全实现为原生C++运行时，支持正反向精确梯度、批处理、发散控制、多核并行，并通过PyTorch autograd.Function暴露为可微张量操作。
- **严谨的实验分解**：通过调优急切基线（payload-only Python）和本地装箱基线，清晰分离了表示改进与原生执行改进，避免了笼统宣称“更快”而无法归因。
- **广泛验证**：随机差分测试覆盖标量算术、控制流、递归等核心语言特性；手工案例涵盖矩阵操作。第二个前端（栈字节码VM）验证了表示的通用性，代码重用仅219行。
- **定量基线对比**：与手写JAX、分阶段图、编译天花板进行定量比较，明确转换点，不回避NDVM的局限。
- **可复现性**：提供完整代码、环境清单、锁定基线命令、参考结果，并展示了每幅图表与脚本的对应关系。

## 8. 不足与局限

- **GPU完整实现缺失**：仅有一个专业前向核（稠密数值上限），完整解释器（包括符号遍历、堆、磁带、梯度）在GPU上未实现。论文指出当前CPU引擎在控制流密集、小算术负载下是合适的，但高维应用需未来工作。
- **梯度正确性仅为实证**：Property 1（轨迹常数梯度）声明为预期语义，但未给出形式化操作语义或机器验证证明。验证仅通过经验性前向/梯度等价（与基线后端比较）和有限差分检查，但未涉及形式化证明。
- **语言支持有限**：不支持突变（堆为一次性写入）、异常、一等延续、动态形状、效果I/O。差分测试仅覆盖标量表面，未包含矩阵形状、列表、闭包的随机生成。对象语言可能超出NDVM支持构造。
- **发散控制可能降低批处理收益**：当种群中控制流高度发散时，批次掩码导致部分通道闲置，批处理乘子从~

当种群中控制流高度发散时，批次掩码导致部分通道闲置，批处理乘子从约60x（完全一致控制流）下降到约1x（完全发散）。这意味着在高动态、每通道独立分支的应用中，批处理收益会显著退化，NDVM 的优势可能消失。

- **场景外推有限**：所有实验均基于特定前端（Scheme 子集或栈字节码VM）和人工/LLM提议程序。在更复杂的语言（含一等函数、动态形状矩阵、可变数据结构）或更大规模搜索空间（如神经网络架构搜索）中的表现尚待验证。论文明确指出矩阵重载任务和在线模型提议是未来工作。

- **单CPU核上限**：尽管多核扩展接近线性，但当前实现完全限定于CPU。GPU虽然可承载有效载荷的数学运算，但结构遍历、堆管理、磁带记录等控制器逻辑难以高效映射到GPU线程模型，论文未提供完整的GPU实现，仅给出了一个专业前向核的数值上限作为参考（RTX 4090上约40–140× 相对于单核CPU纯数学部分），但这不反映完整解释器性能。

- **编程复杂度**：NDVM 的实现（约3000行 C++）虽比从头编写JAX前向/梯度对简单，但比使用现成可微框架（如PyTorch）的 eager 模式更复杂，需要维护一个独立的解释器栈。对于仅需少量候选程序的场景，引入NDVM带来的实现成本和维护负担可能超过收益。

- **前端适配开销**：要将新语言或新后端接入NDVM，需要编写一个编译器将源程序编译为NDVM指令集（VCode）。论文提供了第二个前端（栈字节码VM）的示例，但通用化仍需额外工程工作，不像自动微分框架那样即插即用。

## 9. 未来工作

论文指出了几个明确的后续方向：

1. **GPU完整实现**：将NDVM的符号遍历和磁带管理迁移到GPU，或设计混合执行模式（CPU控制流 + GPU数学核心）。需要解决分支发散、堆周转、闭包序列化等挑战。
2. **支持更多语言特性**：添加可变数据结构、异常处理、一等延续、动态形状矩阵等，以扩展NDVM的适用场景。
3. **在线模型提议**：将LLM（或其他程序生成器）纳入搜索循环，并测量整体端到端加速，而非仅内核校准时间。
4. **自动优化编译**：利用结构化缓存或部分求值技术，在多次重用同一程序骨架时自动切换到分阶段图模式，使NDVM能在低重用和高重用场景间自适应切换。
5. **形式化正确性**：为NDVM的梯度语义建立操作语义和形式化证明，或使用证明助手（如Coq）验证Property 1。

## 10. 整体评价

该论文提出了一种新颖的运行时表示策略——区分评估器而非程序，从根本上缓解了神经符号联合搜索中“程序即数据”与高效率执行之间的矛盾。通过将离散结构与数值有效载荷分离、并编译一个固定的可微评估器，NDVM实现了在保持搜索灵活性的同时，前向和反向时间显著降低（单核上最高2000倍以上），批处理和多核扩展也接近理论极限。实验设计严谨，分解清晰，对比充分，代码公开，具有很高的可复现性。

尽管存在GPU实现缺失、语言支持有限、发散场景批处理退化等不足，但该工作为可微解释器领域提供了一个强有力的系统级解决方案，并展示了在标量回归和递归重载等联合搜索任务上的实际加速效果（吞吐量提升20–340倍）。对于需要频繁评估大量结构不同但数学运算轻量的程序的场景（如符号回归、科学发现、程序合成），NDVM是一个极具实用价值的基础设施。
